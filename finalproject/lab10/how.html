How is it Used?

Micropub enables users to interact with their websites using third-party applications. Below is a step-by-step explanation with examples:

  1. Connecting the Client to Your Website

    Example:

    A Minipub client (like the Indigenous app or Quill) communicates with your website's Micropub endpoint. If you have a WordPress site with the Micropub plugin installed, you log into the client by entering your WordPress site's URL, securely connecting the client to your website.

  2. Sending Requests

    Example:

    You write a draft blog post on the Quill app. When you click Publish, Quill sends a message called a Micropub POST request to your site. If you decide to edit a previous post, Quill sends an edit request with the post's ID and the updated content. If you want to delete, it sends a delete request.

  3. Processing Requests

    Example:

    The Micropub plugin on your WordPress site processes these requests:

    • The plugin processes the action:
      • Create: Adds a new blog post to your dashboard.
      • Edit: Updates the specified post with new content.
      • Delete: Moves the post to the trash or permanently deletes it.
  4. Displaying the Updated Content

    Example:

    Your website updates its database and displays the post on your blog.
